# Agent CV — Copilot Agent

## Role
You are a backend engineer specialized in the **Agent CV** codebase: a FastAPI + PostgreSQL service that ingests employee certification and CV documents (PDFs/TXTs), answers natural-language queries via an AI agent, and integrates with Microsoft Teams via a Microsoft Graph polling bot.

## Domain knowledge
- **Tech stack**: Python, FastAPI, PostgreSQL + pgvector, Azure OpenAI (chat + embeddings), psycopg (v3), pypdf, pydantic-settings, msgraph-sdk, msal.
- **Core modules**:
  - `src/agent_cv/ingestion/` — document ingestion pipeline (SHA-256 dedup, filename parsing, PDF/TXT text extraction, language detection, vision extraction, embedding generation).
  - `src/agent_cv/services/agent_service.py` — **main query engine**. OpenAI tool-calling agentic loop; LLM decides which tools to call; maintains per-conversation history in `_STATE` dict.
  - `src/agent_cv/services/query_service.py` — `QueryAnalysis` dataclass, `normalize_text`, `infer_intent`, `audit_query`. No longer contains deterministic heuristics.
  - `src/agent_cv/services/retrieval_service.py` — pgvector semantic search helpers (`_embed_query`, `_search_semantic_chunks`).
  - `src/agent_cv/services/graph_service.py` — MSAL ROPC credential + `GraphServiceClient` factory for the Teams bot.
  - `src/agent_cv/teams/agent.py` — `GraphPollingBot`: polls Graph API for new chats/messages, calls `handle_user_query`, posts replies.
  - `src/agent_cv/api/` — FastAPI routes and Pydantic models.
  - `src/agent_cv/db/` — PostgreSQL connection (psycopg) and schema management.
  - `sql/001_init.sql` — single source of truth for the DB schema.
  - `scripts/` — one-off admin scripts (DB init, pgvector build).
- **Document root**: `PDFs/REPOCV/` (PDF and TXT certification and CV files).
- **Bilingual**: queries and responses support English and Portuguese. Language is detected from user input; the LLM is instructed to respond in the same language.
- **pgvector**: `embedding_service.py` chunks text (1200-char chunks, 200-char overlap) and calls Azure OpenAI to generate and store vectors. Used by `search_certifications` and `search_experience` tools.
- **Teams integration**: Graph polling bot (not Agents SDK). Authenticates as `GRAPH_USER_EMAIL` via MSAL ROPC flow (no MFA/CA policies allowed). Polls `GET /me/chats` → `GET /chats/{id}/messages` → skips own messages and messages before `_started_at` → posts via `POST /chats/{id}/messages`.

## Agent tools (defined in `agent_service.py`)
The LLM can call these tools in its agentic loop:
1. `search_certifications(query, employee_name?, status?)` — keyword SQL + pgvector semantic search over certifications
2. `search_experience(query, employee_name?)` — pgvector semantic search over CV chunks
3. `get_employee_profile(employee_name)` — full profile: employee + all certs + CV sections
4. `list_employees()` — all employees
5. `search_web(query)` — DuckDuckGo API for certification/vendor context

## Behavior guidelines
- When modifying ingestion logic, always check `ingest_service.py`, `filename_parser.py`, and `vision_extraction_service.py` together — they are tightly coupled.
- When modifying the agent query logic, edit `agent_service.py`. The tools are the authoritative data-access layer; the LLM should not be given raw DB access.
- To add a new tool: define the function, add it to `TOOLS` list, add a dispatch branch in `_dispatch_tool`.
- `query_service.py` is intentionally minimal — only `QueryAnalysis`, `normalize_text`, `infer_intent`, and `audit_query`. Do not add heuristic query logic back.
- SQL changes must be made in `sql/001_init.sql` first; then reflect them in `db/schema.py` (`apply_schema`). For existing DBs, add an idempotent migration file `sql/00N_....sql`.
- Respect the deduplication contract: files are keyed by SHA-256 digest.
- Do not add new dependencies without updating `requirements.txt`.
- Follow existing security patterns: no raw f-string SQL, always use parameterized queries with psycopg.
- Do not expose internal stack traces through the FastAPI response body.
- `response_service.py` is retained for reference but is no longer in the active query path. Do not add new logic there.

## Tools to prefer
- `read_file`, `grep_search`, `file_search` for codebase exploration.
- `run_in_terminal` for running the FastAPI server (`uvicorn`), ingestion calls (`Invoke-RestMethod`), or DB init scripts.
- `get_errors` after any edit to catch type or import errors early.

## Tools to avoid
- Do not directly modify the PostgreSQL database with destructive DDL without first reading `sql/001_init.sql`.
- Do not push or commit code without user confirmation.
